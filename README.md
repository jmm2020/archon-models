# archon-models

A small, focused **workflow router** for [Archon](https://github.com/coleam00/Archon) — the open-source harness builder for AI coding agents.

Given a natural-language user request, the router picks the single best Archon **workflow** to run (or abstains and escalates when nothing fits). It is a **LoRA adapter** fine-tuned on `unsloth/gemma-4-12b`, trained locally with Unsloth + TRL.

> **Status: v1 baseline / pipeline-validated.** The training pipeline runs end-to-end on a single GPU and the adapter converges, but v1 is trained on a small, canonical-anchor dataset. Treat it as a working starting point, not a production router. The honest data story — and how to make it good — is below.

---

## Why this exists

Archon defines development processes (plan → implement → validate → review → PR) as YAML workflows. As the number of workflows grows, *choosing* the right one for a request becomes its own task. This repo is an experiment in training a **narrow, deep, local** specialist to do that routing cheaply and offline, instead of spending a frontier model's context on it.

## What's in here

```
  seeds/phase0_raw_extracts_v2/  # raw extracted anchors (workflow / command / subagent routing)

  archon-router-gemma4-lora/     # the trained LoRA adapter (final, no checkpoints) — Git LFS
examples/

```

## The routing task

**Input:** a user request (natural language).
**Output:** a JSON decision:

```json
{"workflow": "archon-fix-github-issue", "confidence": "high", "escalate": false}
```

When no workflow fits, the model abstains: `{"workflow": null, "confidence": "low", "escalate": true}`.

The label space is the set of routable Archon workflows (`.archon/workflows/*.yaml`, excluding `e2e-*` smoke tests) — 30 routable labels in v1 (see `data/build/build_stats.json`).

## Quick start (inference)

You need the Gemma base model (downloaded separately; accept the [Gemma license](https://ai.google.dev/gemma/terms)) plus this adapter.

```bash
pip install "transformers>=4.45" peft torch
git lfs install && git clone https://github.com/jmm2020/archon-models
python examples/infer_router.py "fix github issue #412 in my repo"
```

See `examples/infer_router.py` for the full load-and-route snippet.

## Reproduce the dataset + training

```bash
# 1. Extract intent→workflow anchors from live Archon YAML (point at your .archon/)
python scripts/extract_routing.py

# 2. Build the train/eval JSONL (Gemma chat format, stratified split + abstention)
python scripts/build_routing_dataset.py

# 3. Train the LoRA on Gemma-4-12B (Unsloth + TRL; --smoke for a 5-step dry run)
python scripts/train_routing_local.py            # full run
python scripts/train_routing_local.py --smoke    # validate the pipeline
```

> The scripts assume a local Archon install (`.archon/workflows/`, `.archon/commands/`). Adjust the paths near the top of each script to your layout.

## Data strategy (read this before trusting v1)

v1 is trained on **canonical anchors** — each workflow's own `Use when:` / `Triggers:` / `NOT for:` text, plus a small hardcoded abstention set (~200 examples total). That's enough to validate the pipeline, but training and evaluating on canonical anchors **measures memorization, not routing skill** — real users don't phrase requests like a workflow's description.

The path to a genuinely good router (the recipe this repo is built to support):

1. **Real run history** — mine actual `user request → workflow that ran` pairs. Perfectly labeled, real distribution. (Highest value; tends to be thin and template-skewed, so it sets label priority but can't train alone.)
2. **Synthesized realistic paraphrases** — 15–30 diverse, natural phrasings per workflow generated by a strong model from its `Use when` / `Does` / `NOT for`. This is the single biggest lever; the `synth/` bucket in `build_routing_dataset.py` is the hook for it.
3. **Contrastive boundary pairs** — every `NOT for: … (use X)` clause names a confusable sibling. Turn those into hard A-vs-B examples; they teach exactly the boundaries a multi-class router gets wrong.
4. **Honest abstention** — coding-adjacent near-misses and genuinely ambiguous→escalate cases, not toy out-of-domain trivia.
5. **Thin retention slice** — keep the base from collapsing into a JSON-only router.

Two principles: keep the dataset **regenerable** (keyed to the *current* workflows, so it rebuilds when workflows change — freeze nothing), and **evaluate on real/diverse held-out phrasings**, never on the canonical anchors the model trained on.

## Model details

| | |
|---|---|
| Base | `unsloth/gemma-4-12b` |
| Method | LoRA SFT (PEFT) via Unsloth + TRL |
| LoRA | r=32, α=32; targets q/k/v/o + gate/up/down proj |
| Precision | 4-bit base, bf16 train |
| Adapter | `model/archon-router-gemma4-lora/` (~525 MB, Git LFS) |

See `model/archon-router-gemma4-lora/MODEL_CARD.md` for the auto-generated card and framework versions.
